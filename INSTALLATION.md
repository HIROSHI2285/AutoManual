# AutoManual インストール手順書 (Windows/Mac対応)

このドキュメントでは、AutoManual プロジェクトを新しいPCのローカル環境でセットアップし、実行するための手順を説明します。

## 1. 前提条件 (Prerequisites)

本ツールを実行するには、以下のソフトウェアがインストールされている必要があります。

### 必須ソフトウェア
*   **Node.js**: バージョン 18.17 以降推奨
    *   インストール確認: コマンドプロンプト(またはターミナル)で `node -v` を実行し、バージョンが表示されること。
    *   ダウンロード: [Node.js 公式サイト](https://nodejs.org/) から "LTS" (推奨版) をインストールしてください。
*   **Git**: ソースコードのダウンロードに使用します。
    *   ダウンロード: [Git 公式サイト](https://git-scm.com/)

### 必要なAPIキー
*   **Google Gemini API Key**:
    *   取得先: [Google AI Studio](https://aistudio.google.com/app/apikey)
    *   ※ 無料プランで利用可能です。

---

## 2. インストール手順

### Step 1: プロジェクトのダウンロード
任意のフォルダでコマンドプロンプト(またはターミナル)を開き、以下のコマンドを実行してソースコードをダウンロード（クローン）します。

```bash
git clone https://github.com/HIROSHI2285/AutoManual.git
cd AutoManual
```

※ Zipファイルでダウンロードした場合は、解凍してそのフォルダ内でコマンドプロンプトを開いてください。

### Step 2: 依存ライブラリのインストール
プロジェクトに必要なライブラリをインストールします。

```bash
npm install
```

※ 完了まで数分かかる場合があります。

### Step 3: 環境変数の設定
Gemini APIキーを設定するためのファイルを作成します。

1. プロジェクトフォルダ直下に `.env.local` という名前のファイルを新規作成します。
2. 作成した `.env.local` をテキストエディタで開き、以下の内容を貼り付けます。
   `your_api_key_here` の部分をご自身の Gemini API キーに書き換えてください。

```env
# AutoManual 環境変数
GEMINI_API_KEY=your_api_key_here
```

⚠️ **注意**: ファイル名は必ず `.env.local` としてください（`.txt` など拡張子がつかないように注意）。

---

## 3. アプリケーションの起動

以下のコマンドを実行して、開発サーバーを起動します。

```bash
npm run dev
```

コマンド実行後、以下のようなメッセージが表示されれば起動成功です。

```
> next dev
- ready started server on 0.0.0.0:3000, url: http://localhost:3000
```

ブラウザ（Chrome推奨）を開き、以下のアドレスにアクセスしてください。

[http://localhost:3000](http://localhost:3000)

### 💡 便利な起動用バッチファイル
プロジェクトフォルダに含まれている `start_app.bat` をダブルクリックするだけで、サーバー起動とブラウザ起動を一度に行うことができます。
デスクトップなどにショートカットを作成しておくと便利です。

## 4. トラブルシューティング

### Q. `npm install` でエラーが出る
*   Node.js のバージョンが古い可能性があります。 `node -v` で確認し、v18未満の場合は最新のLTS版をインストールしてください。
*   ネットワーク環境によってはインストールがブロックされる場合があります。プロキシ設定などを確認してください。

### Q. ブラウザで画面が真っ白になる / サーバーエラーになる
*   `.env.local` ファイルが正しく作成されているか、APIキーが正しいか確認してください。
*   コマンドプロンプトにエラーログが表示されていないか確認してください。

### Q. ポート3000が既に使用されていると言われる
*   別のアプリケーションがポート3000を使用しています。
*   コマンドプロンプトで `Ctrl + C` を押して一度サーバーを停止し、PCを再起動するか、以下のコマンドでポートを指定して起動してください。
    ```bash
    npm run dev -- -p 3001
    ```
    (この場合、アクセス先は http://localhost:3001 になります)
